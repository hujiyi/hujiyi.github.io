coding peasant diary 2022 09 27

发表于2022-09-27|更新于2022-10-11|diary
|总字数:974|阅读时长:3分钟|浏览量:

昨天晚上实在是没有什么事情做, 于是无聊地打开电脑, 手痒去点了更新, 系统提示可以更新到 Windows 22H2, 之前已经下载了这个版本的ISO, 但是没有安装, 实在没忍住, 就选择了升级更新。

更新的时间不算太长, 重启后, 感觉图标要漂亮一些, 其他的没什么弄就关机了。

今天到公司坐了一会儿, 感觉好象没有什么事情的样子, 前面的API差不多写好了, 昨晚打算把注册那个增加一个已报名的默认值, 但打开电脑后, 这个事情其实没想起来, 就想着, 要不要从头开始慢慢搭一个Spring Boot 2 项目的框架。

昨天试了一下 start.spring.io 居然是可以访问的, 不知道是是不给新电脑面子。

今天继续弄, 但是 pom.xml 文件总是报错, 随后 start.spring.io 又无法访问了, 试了几次, 最后用 Dism++ 改了 host 文件, 重新建项目。

还没有搞明白的时候,　家里来电话说, 保姆和女儿外婆出门没有带钥匙, 进不了家门了, 看看时间, 差不多十点了, 于是赶紧下楼, 楼下很显眼的地方已经拉了金融迎接国庆的横幅。

一路走去搭公交车, 还算顺利, 刚到公交车站, 就看到 K118 准备驶入站台。回去开了门赶紧又搭车回公司, 来回花了差不多一个小时。

下午三点左右, 健哥又发来信息要改接口, 就这个, 有时候我觉得, 这好像只是为了让我有点事做而已, 或者说是找一点我能做的东西给我玩一下而已。怎么都好, 那就改吧。

打算把项目 pull 下来, 这时候才发现, 右键菜单 里面已经没有 git 相关的菜单项了, 只好把 git 卸载了再重新安装一次。

改完了继续折腾, 上去看 Spring Boot 2.7.4 的文档, 说是要求 Maven 3.5+ 的版本, 之前一直都用 Idea 2021 自带的 Maven, 具体是多少倒不清楚, 于是就下载了一个最新版的 Maven 3.8.6, 弄好了又想着是不是可以用它来替换 Idea 2021 自带的那个呢? 结果进去一看, 自带的是 Maven 3.6.3, 感觉也是够用的啊。

所以看来前面出现的问题可能是因为选择了 JDK 17, 有些东西大概并不支持这么高版本的 JDK, 好吧, 那就让一步, 再建一个 JDK 11 的项目试试。

基于JDK 11新建的项目没有问题, 后面看子模块中的提示, 要说packaging要求为 pom, 记得之前曾看过的一篇文章说要指定为 pom的, 我前面都没有指定, 所以之前出错有可能是因为这个吧。

新电脑还没有安装数据库, 总的来说, 我不怎么喜欢 MySQL, 所以下载了新版的 Mariadb 10.9.3, 顺便试试 Spring Boot 怎么连接Mariadb, 安装好了, 发现桌面又多了一具 HeidiSQL 客户端的图标, 之前已经单独装过一次这个软件了, 仔细看看路径, 发现这两个图标指向的并不是同一个, 想着要把旧的那个卸载了, 可以到了控制面板的应用里面一看, 不管是新安装还是之前安装的HeidiSQL, 一个都没有, 看来这个也是升级 Windows 11 搞出来的问题吧。

好事多磨, 试了各种组合, Mariadb 可以顺利连接了, 明天吧, 看看是不是可以开始写一个简单的 API 了。

今天走了两趟, 搭了两次公交车, 打卡两次
