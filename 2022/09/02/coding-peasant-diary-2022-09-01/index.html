2022年09月01日 星期四

补写的前面所有的日记, 懒得补每一天, 就干脆补完前面所有时间的吧

之前想着要写一些日记, 感觉还是要一家大公司的产品比较好, 于是在手机上翻出了 OneNote, 不曾想到我的账号已经被禁用, 要登录 OneDrive 解除限制。 然后发现 OneDrive 在国内已经用不了啦, 偏偏连着几天都翻不了墙, 于是日记这个事情就这么拖着了。

2022年08月23日 星期二

今天是到云宝宝 公司报到的时候, 9点上班,但是之前没有云过, 又是搭公交车去的, 所以感觉应该早点出门。

之前从曾经到云宝宝挂职的禤静老师那时了解到, 公司所在的大厦停车费那贵,在网上查了一下, 刚好有26路车可以到达附近, 所以就决定公车上班。 当然也考虑过,如果车站太远的话, 要不要搞一个共享电动车来用用。

早上7点起床, 十分钟左右收拾好出门。过了江北大道往前走, 才发现两个不同方向的公交车站居然隔得那么远。26路到了五象总部基地休闲公园站下车，边了马路， 基本上就可以看到 五象总部大厦了，走到大厦附件, 居然太早了, 于是只好在路边的花圃坐着等。

终于快到9点了, 跟着上班的人走进C座, 搭电梯上到8楼, 807的门开着, 可以门口的牌子是广西世环科技有限公司(照片为补拍), 只有在公司最里面的角落有一个人, 灯也没有开, 黑黑的, 该不会是走错了吧。

沿走走道往相反方向走了一段, 有个洗手间, 刚好可以进去方便一下, 出来了再次回到 807 门口, 犹豫了一下, 走了进去。

问道: 请问这里是 C座 807吗?

里面的人回答: “是啊”

终于踏实了, 没走错地方。

于是自我介绍：“我是南职院的老师，我来公司挂职，今天过来报到…”

……（聊天过程无法还原, 略）

(补拍照片)

很巧的是, 今天最早来到公司的人， 居然是我们软件专业11级软件3班的学生, 他说姓于,名字我没有问, 后面找了去年来这里挂职的赵老师, 才知道叫 于家逸, 打开手机QQ翻看 11级 的班级群, 好吧，当年我主要给2班上课，2班群里的名字都很熟悉， 3班的就几乎没有印象了。

于同学问起了学校的近况，也问了一些他熟悉的老师的情况。

之后不久, 在他对面又来了一个人，一看，很面熟啊。应该是17级的，名字一下不记得了。回头翻了一下通讯录,17软件1班的梁彬, 当年毕业设计他和几个同学一定要选我，可是我没能给他们什么有效的指导, 不过梁同学很冷，不愿意和我说什么，我当时只是下意识地说了一句：你也在这里啊。

过了一段时间， 肖江明 也来到了办公室,先给我按排了一个座位, 过了一下，把我叫到小会议室,说了他的一些设想：既然我在简历上写最近主要做的是跨平台移动开发，那希望我能弄一个 uniapp 的脚手架什么的出来，完善公司的技术架构。当然，在这之前，我还是先要熟悉公司现在进行开发的框架 eladmin-plus。

好家伙，最终还是要和 spring boot 打交道了, 之前已经着手弄了一下这个框架, 至少已经能让它跑起来了，虽然之后就不知道该从哪里下手了。

还是一步步来吧，所以，还是先要了解 spring boot才可以，于是第一天剩下的时间, 就是用 spring boot 做了一点最原始的东西。

中午的时候, 组长说叫 梁彬 带老师去附近吃个饭, 不过那个学生似乎并不想理我, 感觉是因为毕业设计的时候的事情? 于家逸带我去对面吃了午餐, 好贵啊。

收工回家的时候, 发现除了26路， 还有 k118 可以搭, 而且 k118 是快车，站点的数量更少，中间还不绕弯，明天要试一试。

2022年08月24日 星期三

今天决定换 k118路车，上车的站点在白沙桥底那一头，走过去，发现比走到 葫芦顶桥底要近一些。下车的站点就在五象总部大厦的旁边, 比 26路 的站点要近了很多。

早上坐下来不久，出去上个洗手间，离开公司门口，对面走来一个看起来和普通员工很不一样的女人。

回到座位, 组长键哥发来钉钉信息，说等一下去和欧总打个招呼，欧总，估计就是以前填写企业实践表时上面那个叫欧燕的名字。好吧，键哥估计没看到我的回复，直接到座位这里叫了我，而我的座位其实就在欧的办公室门口。

欧总很忙，总在打电话联系业务

总的来说， 欧说问我是不是专心来公司，会不会一下又跑回去上课什么的，象去年的那个老师一样（赵老师），好吧，总要表个决心：我会认认真真在公司做完半年挂职的。

欧总说现在经济不好，以前大家买了房子，房子不仅保值，还升值，房子就是一个定心丸，大家不怕花钱，现在不行了房子不升值，还往下掉价，大家不敢花钱，经济不好。

好吧，道理这样说也通，可是在几十人的办公室说随便买600万的房子什么的，这些领月薪的员工会怎么想呢? 也许大家已经习惯了吧。

快到中午的时候, 于家逸 过来帮我申请、开通了五象总部大厦的食堂云, 自己充值之后就可以到A座的食堂吃饭了。 来到这里两天, 总觉得这里的网络不好, WIFI不怎么样, 手机信息更差, 电信的 5G连不上, 只显示连接了4G, 而且没有信号，手机大部分时间处理没有网络的状态，连着几天都是这个样子。最尴尬的是, 第一次到食堂去吃饭, 明明已经打开了食堂的支付码, 可是死活刷不成功。手机网络就是下图这个状态， 最后 于家逸 用它的手机给我做了个热点，总算才吃上了午餐。

下午的时候，烟厂的吴文权发来信息，说是要帮忙做一个参加区计算机程序设计员赛项的培训, 把上次南宁选拔赛的资料也发给了我，说他公司的同事用Flask 做开发的时候，速度很快, 所以他也打算用 Flask 和 Vue 参加下一阶段的比赛，要我准备好培训他一下。(中间微信语音沟通了，说是培训费可以走对私, 每天有 3K。啊，那我前面6天走学校对公的钱什么时候可以拿到呢? 有没有这么多呢?)

下班前写了一个两天的培训计划，Flask WebAPI和Vue 各一天，计划是写了，可是我觉得Flask还好,Vue 就需要再配上 Element UI，感觉做界面比其他的更麻烦。

6点下班到公交车站搭车， 准备说用手机来支付的，可是继续没有信息, 看来是手机和这个高大上的地方不搭啊。

2022年08月25日 星期四

今天没有心思再弄什么 eladmin 或 spring boot了，之前还写了一个用 spring boot 建项目的笔记, 现在都只能放一边, 先专心把培训的活做好吧。

先是用Flask 做了一个API， 麻烦的地方在于除了 Flask 和 Flask-SQLAlchemy 没有其他的库可以使用了, 那查询结果转为 JSON也要自己写, 更麻烦的是还有一个跨域的问题。最终，不借助第三方的库都能解决了。写好了主要操作的API，看看时间，感觉要还有 Vue 的部分还没有开始呢。于是发信息问了 吴文权, 这个比赛的时间有多长? 回答： 240分钟。 时间明显不够啊。

到了下班的时候，问了为什么要选择Flask和 Vue 的方案参加比赛, 回答是南宁的领队说，比赛技术文件中有提到要选用合适的系统框架，体现分层的软件设计理念， 所以领队说这个分层就是前后端分离，所以要用Vue 做前端。

我觉得这个领队应该不是搞软件开发的人，不清楚什么是分层。当然，我也认为前后端分离也是分层中的一种，而且是分得很彻底的那一种。

手机继续信号不好，无语中。

2022年08月26日 星期五

今天继续写python， 不过我不觉得应该继续搞 Flask 和 Vue， 当然 Flask 的部分已经准备好了。

我觉得还是同时准备一个Django 比较稳妥, 于是今天的时间基本上就是在用 Django 做好一个例子。

一个上午就只完成了一小部分，这240分钟明显不够啊。

最终下班前，完成了 Django 项目的大部分内容

本来应该做 Java 的，但一直专业写 Django， 有一种偷偷摸摸的感觉，心虚。

2022年08月29日 星期一

早上去体检, 下午在家写 Django, PyCharm Community 被阉割了很多功能, 好多代码没有提示, 可是还是能用, 不是吗？边写项目边写 Django 的博客文章。

2022年08月30日 星期二

周六做了一天培训, 最终定为使用 Django，吴文权提了一些要求：做一个和比赛一样的虚拟机环境进行练习, 看看有没有会遇到一些问题。当时不是很在意，在回家安装好环境开始弄的时候才发现，原来和自己平时用的还是很不一样啊

python 3.6.4 下 只能安装 Django3.2.5, 不支持 Django 4，好吧，怪不得以前我下载了一个旧版 Django 程序, 要安装 python 3.6 才能运行。

PyCharm Community 不提供 Django 项目运行的配置, 只能使用命令方式运行调试。

看来, 多次参加比赛的选手就是不一样，这样的问题我以前都是不会考虑的。

一边写 Django 项目代码，一边写相应的博客.

又是偷偷摸摸的一天

2022年08月31日 星期三

这几天给 hexo 博客添加了置顶 和 隐藏文章功能，也学会了使用草稿。把写好的博客文章先发出三篇，后面的代码还没有完善，再改改。

到下班时，Django 项目的基本完工, 好吧，做了这个项目，感觉自己也有不少的收获。

2022年09月01日 星期四

写完了所有的Django 代码 和博客文章, 现在对于 spring boot和 eladmin 已经没有什么感觉了。

早上的时候，组长键哥发给了我一个卫键委人事信息系统 的前后端源代码, 我的天啊，现在需要回到Java这一块来了。

把键哥的代码解压，还原了数据库，配置好项目中的数据库和 redis服务器，项目运行成功。可是我还是不知道怎么用 eladmin plus。 于是再次从 eladmin plus 开始，删掉之前不知道弄成什么样的项目，重新下载一个，从头开始。

我觉得呢，要不要先给项目改个名字? 下载了，直接使用别人的项目感觉不太好吧，于是开始尝试改项目名，改包名，改模块名, 最终，失败。唉，Java 基础不好，难。

下班前，上传最新的 Django 项目代码，发布所有的相关博客文章, 收工。
