使用 PyCharm Community 开发 Django 应用程序 第四部分 - 订单管理

发表于2022-09-01|更新于2023-01-08|django
|字数总计:2.6k|阅读时长:13分钟|阅读量:

开发环境：

  • Microsoft Windows [版本 10.0.14393], locale zh-CN

  • Python 3.6.4

  • PyCharm Community Edition 2021.3.3

  • Visual Studio Code 1.70.0 (system setup), 64-bit edition

当前项目完整代码

订单的模型类定义

订单管理应用的模型类: order/models.py

订单的数据来源于库存 stock , 之前由于考虑不周, 直接使用了商品product

复制模板文件

部分应用的实现与其他应用类似, 没有写详细实现的文章, 具体代码可以看代码仓库

  1. 鼠标选中已经完成的库存模板文件夹 templates/stock, 右键 -> Copy;

  2. 鼠标点击选中项目模板文件夹 templates,右键 -> Parse;

  3. 在弹出的 Copy 对话框中, 修改 New name 文本框内容为: order , 然后 OK

修改订单管理模板

  1. 订单列表模板文件: templates/order/index.html
  1. 订单添加/修改模板文件: templates/order/edit.html
  1. 订单删除确认模板文件: templates/order/delete.html
  1. 订单详情模板文件: templates/order/detail.html

修改视图文件

添加和修改订单时, 可以添加以下两个方面的考虑(具体实现看完整代码):

  1. 添加订单时, 订单商品数量不能大于库存剩余的商品数量, 完成订单时, 同时修改对应库存商品的已售数量

  2. 修改订单时, 要重新调整库存中对的商品已售数量

订单视图文件的完整代码: order/views.py
