使用 PyCharm Community 开发 Django 应用程序 第二部分 - 实现第一个应用

发表于2022-08-29|更新于2023-01-08|django
|总字数:4.8k|阅读时长:24分钟|浏览量:

开发环境：

  • Microsoft Windows [版本 10.0.14393], locale zh-CN

  • Python 3.6.4

  • PyCharm Community Edition 2021.3.3

  • Visual Studio Code 1.70.0 (system setup), 64-bit edition

模型的定义

为减少因为数据迁移可能带来的问题, 最好一次性完成所有模型的定义, 各个应用模型类定义如下:

  1. 客户管理应用的模型类: customer/models.py
  1. 供应商管理的模型类: supplier/models.py
  1. 商品管理应用的模型类: product/models.py
  1. 库存管理应用的模型类: stock/models.py
  1. 订单管理应用的模型类: order/models.py
  1. 用户管理只实现 注册 与 登录 功能, 当前应用中不对 Django 内置的 User 模型进行扩展, 所以不需要定义模型类

定义模板

通过模板继承可以可以减少网页中编写的代码, 更容易实现网站整体风格的统一, 模板文件可以分为用于布局的模板和用于某个应用内特定功能的模块

布局页模板

在 templates 文件夹下添加以下三个 HTML 文件:

  1. layout.htm: 用于网站的整体布局, 导入Bootstrap3 所需要的静态文件
  1. nav.html: 导航栏
  1. pagination.html: 通用的分页组件

PyCharm Community 在编写模板文件时, html 代码会智能提示, 但 模板语言 所使用的代码没有智能提示, 部分 HTML 与 模板语言 代码结合时, 虽然代码是正确的, 但还是可能会出现表示语法错误的红色波浪线

适用于查改增删等功能的应用模板

在 templates 文件夹新建名为 customer(与应用同名) 的新文件夹, 在 templates/customer 文件夹中添加以下四个html文件

  1. index.html: 用于显示客户列表数据列表、提供各增、删、改等功能连接或按钮
  1. edit.html: 添加与修改功能共用的模板文件
  1. delete.html: 用于删除数据前进行确认的模板
  1. detail.html: 用于显示记录详细内容的模板
