对20级软件专业的同学谈谈有关实习的体会

时间过得真快, 转眼间我们 20级 的同学准备到出去找 实习工作 的时候了。

结合我以前指导实习、就业, 还有现在我在云宝宝公司挂职了解到的一些情况, 谈一下我的一些看法：

我所在的部门的情况

我这个学期到云宝宝公司挂职, 从8月下旬开始进入公司的 业务部, 整个部门有30多人, 部门的技术负责人是我们软件专业 06级还是 07级的学生（当时聊完了后来忘记具体是哪一级了）。

另外部门里面还有一个是 11级软件3班和一个 17级软件1班的同学。所以如果大家以后在南宁市从事软件开发方面的工作, 有很大的可能, 你都会遇上软件专业的学长。

每天上班时间是上午九点到十二点,下午两点到六点, 当然在公司项目进度吃紧的时候, 其他人还是有时有需要加班的情况(我是编外人员,所以每天准时收工走人)。

在和公司业务部联系之后, 他们就给了我一个业务部现在正在使用的项目框架（Spring Boot 2 + Vue 2）。因为我之前是基本不使用 Java, Spring Boot 也是没有用过的, 这个框架我花了差不多两个星期才用起来(因为Java 基础不好,所以有不少地方是知其然而不知所以然), 现在的项目都是在这个框架的基础上进行开发的。

我所在的项目小组的情况

到公司之后我被分到一个开发小组(项目二组), 连我一起总共5个人, 小组现在主要有两个项目: 一个是已经上线的旧系统,目前对方正在提出一些项目的优化要求,需要进行旧项目的修改和优化; 另外一个是新项目, 目前第一阶段的后端开发工作已经基本完成,主要的工作都集中在前端开发方面,据说要争取在月底前上线项目的第一阶段功能。当然, 大多数时候我都在打酱油, 只是做一些不是很难的部分。

项目开发的一般流程：

部门中有专门的人(类似项目经理, 也许就是吧, 我没有问过她具体的岗位)负责和项目的客户联系, 各个不同的项目都是由她负责管理的, 由她联系并落实项目的各方面需求, 之后她会根据需求做出 项目原型 (项目原型使用 https://www.mockplus.cn制作), 项目原型的界面和一些主要的交互都会和客户确定落实。

我们小组的组长根据需求先确定并建好数据库的表和字段, 接下来就是创建项目的后端部分, 为了提高速度和效率, 后端一般是先使用代码生成器, 再根据需要对部分生成的代码进行少量修改。虽然我没敢问, 但很大的概率是这个项目能够先跑起来就行, 至于代码质量或是更高阶的要求什么的, 看起来是没有时间处理了。接下来的工作主要就是前端部分, 前端开发就根据后端生成的 Swagger 接口 还有 项目原型 进行开发,根据客户的需要, 现在前端项目一般都有 Vue 或是小程序等多种类型。

总的来说,整个项目的开发过程中,前端的开发工作量比后端要更大,我们小组中有一个人是专职做前端开发的,另外一个重点做运维, 其他的人除了要做一些后端的工作, 也要帮忙、协助做一些容易的前端部分的开发。

我理解的一般技术公司招聘实习的需求

公司招人的主要目的是干活, 而不是招一个什么都不会的人来学习的,当然培训机构例外,培训机构后面另外说。因为从公司的角度,只有项目吃紧、人手不足的时候才有招人的动力, 没活干, 招几个人过来发工资还要教会各种技能这个事情是不可能的。 所以招人的总原则是能干活, 或者是能在尽可能短的时间内了解并掌握公司当前正在使用的开发工具, 然后上手干活。因此, 有打算要应聘或是寻找开发岗位工作的同学, 就要自己先做好相应的技术准备。

学校里面教的东西不可能满足外面所有公司的技术需求, 但所有的技术其实也是有相通的地方的,所以大家能把现在的课程学好, 即使以后工作用到的技术并不一样,也可以在更短的时间内掌握上手。招人干活的公司会知道他们自己现在所使用的技术是什么情况, 实习生是马上能上手, 还是要经过一段时间的学习。但是抱着到时了公司再学习的心态去应聘, 基本上成功的概率较低; 但是自己能独立做一些东西, 也意味着你接受其他的知识会更快, 那对公司来说花费的时间以及其他成本就会更低, 也更容易从中招到自己需要的人。

找实习工作要做哪些准备

就我目前做的部分项目来看,后端开发部分是先建数据库,然后用框架的代码生成器生成主要代码(这个要看每个公司的情况, 有些公司可能用的是其他开发框架, 纯手工写代码的也是有的), 最后根据业务的需要对其中部分后端代码进行修改, 修改的主要就是某些表的 查改增删 或是多表之间的关联等内容, 现在我做的项目数据库的表之间都不用外键, 复杂的多表查询调用SQL完成, 简单一些的就在 Java 中的 Service 层完成, 更简单的就直接在前端中分别查询再组合。 前端开发的就目前很多公司主要还是使用 Vue2+Element UI(如果使用的是 Vue 的话)。

就难度来说,个人觉得后端开发的部分和学校开设的课程可能更加衔接,毕竟我们在 Java 方面开设的课程的数量 也更多。

前端部分因为课时等关系, 我之前上这个课的时候,内容并不深入(已经有同学说：谁说不难,像天书好嘛！),而在实际前端项目开发的时候,状态管理、混入等没有讲过的内容都是项目开发不能缺少的部分。至于小程序 的话,其实总的结构 和 Vue 是差不多的。熟悉 Vue之后也能很快上手小程序 了。

如果我们同学打算应聘软件开发岗位的工作,那么就要做好相应的准备：后端开发应该能用 Spring Boot 做一个简单的网站或是 API（当然越复杂的越能体现自己的水平,不是吗？）; 相对来说,前端开发目前用的比较多的是 Vue 和 React, 当然我们开设的课程是 Vue, 所以要侧重前端开发的同学,应该是把 Vue2 + Element UI 搞熟,如果还能会点小程序什么的,估计那就更吃香了。

当然,从我和一些往届同学了解到的情况来看, 现在更多需要前后端都会的全栈开发,不过如果你做不到全面,现在应该把其中一方面弄好,另外的多少也要了解掌握一些, 实现上现在是前端开发后端化,你掌握一端之后,另外 一个其实也不会难了。

现在开始学习还来得及吗

有同学说,现在学习还来得及吗？ 学习什么时候都不算晚! 我们软件专业开设一门课程,总学时就是 60-70左右,每学时40分钟,所以一门课也就不到 3000分钟的学习时间,大概就是60个小时,如果能每天花四个小时学习,大概就是15天。

当然了,学校里面还有老师上课,所以很多同学学习一门课程、甚至自己做练习的时间,也就更少了。这么少的时间,估计用来打游戏,还没有能够上手,何况是学习呢？所以那些抱怨自己没学会的同学,反思一下,自己在这个课程上到底花了多少时间做练习,学习不是花听老师讲,看老师做练习,编程的学习是看自己做了多少的练习,解决了练习过程中遇到的多少问题。你的编程能力和解决过的问题多少成正比,光是看视频 的时间以及看老师演示是没有太大意义的。

所以要是专心去做的话,现在开始补课,还是来得及的。除了上课的教材,网上也有不少的教程资料, B站上也有很多培训公司发面的免费学习视频或是教程,有问题多问会的同学或是老师,如果能在一个课程上投入一到两个星期的时间专心练习,你会有收获的。

去培训公司可行吗？

如果有一家公司招聘的时候,说没有任何基础都可以,包教会,然后再安排工作,那么基本上可以确定这是一个培训机构。

前几天在公司的群里,某小组负责人号召大家给他介绍实习生,说他们那里的项目任务很重,实习来到了可以快速成长。所以,如果你是老板,招一个人来公司,光学习,还给发工资,你会做吗？

现在的培训公司基本上都已经不认真教技术、教学习了。

目前的培训公司主要的套路就是跟一些贷款公司合作,让你签一个协议,把钱贷出来给他,然后你自己还那个贷款,他们的核心工作都放在怎么让你贷款这个事情 上面的。

很多培训公司并不愿意认真教技术了,因为那个周期长效益不好。他们更加喜欢让学员背一些面试技巧,如果你用这些技巧通过公司面试,那他们就完成了协议,就算完不成,贷款的人是你自己,公司早就已经把钱装入口袋了,到时候贷款公司来找的人也是你,而不是培训机构, 这个事情是14级的某个同学找工作时候的亲身经历, 当时他一直和我联系, 和我探讨找工作遇到的各种问题。 最终他通过了面试,但实习期过了没能留下来 。

去面试开发岗位的实习要准备什么?

我没有参加过公司的面试, 之前和部门中 11级的那位同学聊过, 他说自己以前在学校的时候学得也不怎么好,出来了好长时间也找不到合适的工作,后来也是自己静下心来认真地钻研,然后做了一个小项目去参加面试才进入第一家开发公司的。按照11级学长的说法,参加这一类面试,有作品说话是最好的：做一个查改增删的小项目,把项目部署到网上(如果可以的话), 那些公司负责招聘的人看到这类项目,并询问一些相关问题自然会知道我们相关知识的能力如何。

实习生并不需要全能, 但公司需要一个能干活的人、或者说需要一个能更快上手干活的人, 自己做过若干个小项目,自然是很有说服力的一种方式。

当然,有同学会问,那我这个项目要做到什么程度？ 尽自己的能力, 从简单的开始,做到自己能做的最复杂的程序。 几个表的查改增删这些总是需要的吧。其实你能做出基于两个、三个表的项目,那再多几个表其实也不是什么很难的事情了。

总的来说大概就是：你自己能独立写一点功能相对完整的小项目(其实这也是我上课的时候、考试要求的), 如果能够把几门相关课程的知识结合起来那就会更好, 比如 Java 开发的后端、Vue 做的前端, uniapp 做的移动端或小程序或 Flutter 等。 或是基于某些 API 做的前端、移动端什么的。

假如你还不能独立写一些代码,去面试开发岗位是有些困难的,或是面试成功了,到公司之后使用相应的开发框架也要花费更多的时间和精力,最终付出的学习成本并不会少。

当然了,非软件开发的岗位我就不提了,不了解,也不好说些什么。

最后

毕业设计展之后大家就要出去找工作了,希望有志从事软件开发工作的同学,在这段时间里想好自己要做什么,有时间的话,认真补一下自己的不足的课程还来得及, 实在不行,可以从现在到过年的这段时间认真补,时间还是足够的,最后祝大家都能找到一个满意的工作。
